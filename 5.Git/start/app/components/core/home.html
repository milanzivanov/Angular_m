Assignment

Make sure you read the About page before starting your assignment.
Your assignment will be to take what we've learned today and apply it to your project.
Important: if you have completed all the non-bonus excercises from the last class then it is highly recommended that you take your project and work with it instead of this one. For your convenience copy the assignment from below into your home.html file. If you haven't finished all of the non-bonus excercises use this project instead.
Note: use bower to install all the needed libraries.

Complete the following exercises:
  1. Setup your project on a git remote repository (Note: if you're using windows make sure you install GitBash before proceeding):
    1. Create a profile on GitLab, BitBucket or GitHub (Note: GitHub offers only public repositories);
    2. Create a new repository and set the name to something meaningful, i.e. CompanyRegistry;
    3. Open the terminal in the root folder of your project, or even one level above that and initialize a local git repository (git init)
    4. Bonus: read about gitignore and add the dist, assets/js and node_modules folder to be ignored;
    5. Add all the project files and folders to the staging area (git add), excluding the ones that your tools (npm, bower, gulp) generate, i.e. node_modules, assets/js and the contet of the dist folder;
    6. Perform a commit on your master branch (git commit -m "Project init")
    7. Push to your remote repository (Note: you will be asked for credentials so read the terminal and follow the instructions);
    8. Create and switch to a new branch called "development" (git branch development; git branch checkout development);
    9. All code you write from now on will be commited to this branch and only when you are sure you've created a whole component and tested it completely will you checkout your master branch and merge the new features from your development branch;
    10. Bonus: create a new branch every time you start developing a new component or feature, after which you will merge the changes into your development branch and delete the component branch;
    11. Bonus: create a conflict and resolve it by creating two branches, editing the same file (and the same code) on both branches and then merging the changes into the development branch from both branches;
  2. Using the localization examples (2 and 3) extend your application to provide support for i18n. Think about where the code to register and configure the dependencies should be placed and where you will write the translations for the labels:
    1. Localize every string label in the employee.html and employee-list.html file, including the string you get from the router;
    2. Install angular-dynamic-locale and set the default language for both angular-dynamic-locale and angular-translate to English;
    3. Extend your header to provide buttons/links/images that changes the language of the user interface on click;
  3. Use the angular-ui-bootstrap library to extend your application using the following components:
    1. Expand the employee add/edit form so that there is a field named "dateOfBirth" and use the datepicker component for that field;
    2. Expand the add/edit form of the company by adding a "Add new place" button next to the Headquarters combobox. By clicking on the button, a modal window should appear with the name and postal code field for creating a new place. By adding a new place make sure to both add the new place to the list of places in the controller and save the new place on MongoLab;
    3. Bonus: expand the add/edit form of the employee to provide the same functionality for adding new places (Note: since both employee and company use the same modal window, consider where to place the modal template and controller to avoid code duplication);
    4. Bonus: transform the localization buttons to form a dropdown menu using ui-bootstrap;
  4. Change the code of all the combobox components to use the ng-option directive instead of ng-repeat (Note: you'll notice that this breaks the search by place of birth functionality so avoid changing that combobox);
  5. Bonus: when deleting places add the following functionality:
    1. Bonus: If the place is referenced somewhere, i.e. there is a company which has the headquarters field set to that place don't allow that place to be deleted and use the modal dialogue to notify the user that the place is referenced (Note: you will need to analyze the server response to see that the headquarters attribute of a company is actually the name of the place. Use this information to form a request to the server and see if there is a company with the given headquarters);
    2. Bonus: If the place isn't referenced anywhere display a modal dialogue which asks the user if he is sure he wants to delete this place;
    3. Bonus: Provide this modal dialogue check when deleting employees and companies and avoid code duplication;
  • Focus on doing non-bonus excercises during class;
  • What you don't complete during the class remains as a homework assignment;
  • It is highly recommended that you complete each of the listed excercises, including bonus excercises;
